# Strategic Positioning: Markdown-First Gym Primer

## Project category

GymPrimer is an open-source Markdown exercise-literacy primer for gym beginners.

## Primary user

Adults in their first ninety days of regular gym training, or returning after a long break, who need to understand common exercises before following a workout plan.

## Primary pain

Beginners often stand in front of unfamiliar equipment without knowing what it trains, how to set it up, what the movement should feel like, what mistakes matter, or when to stop.

## Primary promise

GymPrimer helps beginners read a short, plain-language Markdown page and leave with a safer, more accurate mental model of one exercise or training principle.

## Core mechanism

The source of truth is a repository of self-contained Markdown pages with disclaimers, conservative scope, claim-level citations for safety guidance, per-page sources, and optional original diagrams.

## Alternatives

The project is different from short-form fitness video, exercise databases, workout apps, AI fitness chatbots, and trainer-only programming tools.

## Tradeoff

GymPrimer chooses a small, citation-backed Markdown corpus that can be read and reviewed in GitHub over a richer app, generated public package, formal expert-review lifecycle, or broad exercise catalog before the content format is proven.

## Compatibility surfaces

Repository paths, stable Markdown URLs, citation conventions, license terms, media provenance, future locale paths, optional mdBook output, and archived historical artifacts are compatibility surfaces.

## Refusals

GymPrimer refuses workout planning, medical advice, injury diagnosis, rehab protocols, AI coaching, video-first instruction, advanced lifting instruction in the first slice, and external media whose rights cannot be verified.

## Falsifiability

The direction fails if beginners cannot use the first five pages to explain purpose, setup, steps, and stop conditions; if safety claims cannot be traced to sources; if the repository is hard to browse as the product; or if the project drifts back into platform machinery before proving the Markdown cards.
