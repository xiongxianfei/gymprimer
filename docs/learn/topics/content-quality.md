# Content Quality Lessons

## 2026-06-29: Responsible Breadth pages need operational understanding, not only structural compliance

- Source session: `docs/learn/sessions/2026-06-29-responsible-breadth-content-quality.md`
- Primary classification: durable-lesson
- Secondary routes: future proposal/spec/template amendment for pattern-page contract split

Structural page contracts are necessary but not sufficient. The anterior-pelvic-tilt page improved when it moved from a short template-compliant explanation to a page that gave the reader a working definition, neutral comparison, observation-not-diagnosis cues, inline red flags, stronger evidence uncertainty, a necessary visual, professional routing, and next-page links.

Use this pattern for future Responsible Breadth pattern and condition pages:

- Start with a concrete working definition using landmarks, motion, position, or equipment parts.
- Give the reader a comparison point so the concept is not floating in abstract prose.
- Add observation cues, but label them as observation rather than diagnosis.
- Inline topic-specific red flags before linking the general red-flags page.
- Make the uncertainty section specific and evidence-backed.
- Use institutional and peer-reviewed sources where health-adjacent claims are central.
- Add necessary visuals when the concept is visual; prefer original SVG for simple diagrams.
- End with real next-page links, and record missing exercise pages as follow-ups rather than adding dead links.
- Update manual proof records and rerun validation after semantic content edits.

This is curated guidance, not a page-contract change. If a section such as "How to notice this in yourself" should become mandatory, update the relevant spec and template through the normal workflow.
